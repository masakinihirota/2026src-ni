## Design Context

### Users
- Primary users are residents and prospective residents of 日鋼団地 using smartphones and PCs.
- They need to quickly understand room availability pressure and submit ranked room preferences without in-person multi-day selection meetings.
- Core job: select north/south building rooms, rank top 3 preferences, and see overlap pressure clearly.

### Brand Personality
- Personality: practical, clear, reassuring.
- Emotional goal: reduce anxiety and confusion during room selection by making choices and crowding visible at a glance.
- Tone: plain Japanese, low-friction guidance, no ornamental wording.

### Aesthetic Direction
- Mobile-first, task-first interface with strong legibility and explicit state feedback.
- Keep existing room-size color coding fixed; visualize overlap using additive signals (badges, border emphasis), not color replacement.
- Web app UI focused on operational clarity over decorative expression.

### Design Principles
1. Decision clarity first: every screen must reduce uncertainty about “which room to pick next”.
2. Preserve domain visual language: room-size colors are immutable and must remain readable.
3. Minimize interaction cost: direct map interaction + synchronized form controls + drag-and-drop ranking.
4. Keep status explicit: always show current saved state and update timing.
5. Prefer consistency over novelty: predictable patterns for residents with mixed digital literacy.
