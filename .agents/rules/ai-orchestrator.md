# AI Orchestrator Guidelines

このファイルは、Antigravity およびその他の生成AIエージェントに対する最上位の指示書（オーケストレーター）として機能し、`.github/copilot-instructions.md` と同等の方針を `.agents/rules` 用に定義したものです。
AIエージェントは、このファイルに定められたルール参照の優先順位と役割分担に従い、既存の GitHub Copilot と共存・協力してタスクを実行します。

## ルール配置と参照方針

- 本ファイル（`ai-orchestrator.md`）および `.github/copilot-instructions.md` は最上位方針のみを扱います。
- コーディングルールの詳細は `.agents/rules/` 配下の各専用ファイルで管理されます。
- AI は実装前に、機能や要件に対応する `.agents/rules/` 配下の関連ルールを必ず参照してください。
- `.agents/rules/` と `.agents/skills/` が矛盾する場合は、`.agents/rules/` を優先します。

## 参照優先順位（衝突時）

1. 本ファイル や `.github/copilot-instructions.md`（最上位方針）
2. `.agents/rules/*`（実装ルールの正本）
3. `.agents/skills/*`（作業手順・ベストプラクティス）

## プロジェクトの概要

このプロジェクトは、`masakinihirota` Web アプリケーションを開発するためのマルチリポジトリワークスペースです。
以下の関心を分離したリポジトリで構成されます：
- **`vns-masakinihirota`**: コアアプリケーションコード（実装・テスト）、生成AI用の指示書、ドキュメント。
- **`vns-masakinihirota-design`**: 人間が作成する設計書・構想・要件定義。

**調査等に関する方針**:
- リサーチ・調査のようなタスクは積極的にサブエージェントに任せて、メインエージェントの処理コンテキストを保護します。

## Design Context

UI デザインは以下の原則に基づいて進行します。

### Users
- **機能**: 価値観やプロフィールを整理・入力する（ITに不慣れな層や高齢者を含む一般ユーザー）。
- **利用状況**: ブラウザ上で段階的にプロフィールを入力・作成し、作品・価値観・スキルを登録する。
- **目的 (Job to be done)**: 自分の嗜好と価値観を分かりやすく記録し、後続のマッチング・表示で一貫して使えること。

### Brand Personality
- **3語**: 信頼感 / 知的 / 親しみ。
- **トーン**: 丁寧で平易。威圧せず、専門用語を避けて行動を促す分かりやすい表現。
- **感情目標**: 第一に「安心感・信頼」。加えて「迷わない明瞭さ」。

### Aesthetic Direction
- **基調**: 単色ベース（グラデーション装飾は禁止）。コントラスト・余白・境界線・タイポグラフィによる情報階層の表現。
- **カラー**: ネイビー系（主軸）、落ち着いたゴールド系（アクセント）。
- **テーマ**: Lightは Glassmorphism、Darkは Elegant。
- **レイアウト参考**: X (旧Twitter) のような3カラム構造（左側の機能一覧ナビゲーションなど）の方式を取り入れる。
- **アンチパターン**: 「AIらしさ全開」のジェネリックな生成AIライクの見た目は厳格に避ける。
- **タイポグラフィ**: 最小16px、行間1.5以上で可読性を最優先。
- **アクセシビリティ**: WCAG AA準拠、reduced motion配慮などの色・動きへの配慮。

### Design Principles
1. **まず理解できること**: 用語より意味。短い補足語で文脈を明確化する。
2. **まず読めること**: 16px以上、十分なコントラスト、詰めすぎない適度な余白。
3. **まず迷わないこと**: 1画面1意図。主要アクションは視線導線上に配置。
4. **まず安心できること**: エラー・状態変化は平易な日本語で次の行動を示す。
5. **まず一貫していること**: 色・余白・ボタン・命名を画面横断で統一する。
