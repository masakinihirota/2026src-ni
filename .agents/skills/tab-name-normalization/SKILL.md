# Tab Name Normalization

タブ名を扱う実装で、正規化と重複禁止を安全に横展開するためのスキルです。

## 1. このスキルを使う場面

- `/work` `/values` `/skills` などでタブ名管理を変更するとき
- タブ名の初期値を変更するとき
- 同名タブの生成を禁止したいとき
- 追加と編集で命名ルールを統一したいとき

## 2. 実装ルール

1. 正規化: `trim()` 後の値を正本として扱う。
2. 空文字: 追加/編集とも拒否する（編集は元の名前維持）。
3. 重複: 追加/編集とも拒否する。
4. 判定ロジック: 追加処理と編集処理で同一関数を使う。
5. UI層: エラーハンドリングや表示制御のみ。判定本体はロジック層に置く。

## 3. 最小テストセット

- 初期名テスト
- 追加時重複拒否テスト
- 編集時重複拒否テスト
- 空文字編集維持テスト

## 4. 運用フロー

1. まず `.agents/hooks/tab-name-normalization-check.md` のチェックリストを実行
2. ロジック層を修正
3. 画面側の反映処理を調整
4. 変更点直結テストを実行
5. `pnpm build` を実行

## 5. 完了定義

- 追加と編集で重複禁止が一致している
- タブ名の初期値が仕様どおりである
- 直結テストが成功している
- ビルドが成功している
