# Test Case Outline Generation - Agent Instructions

## 目的

TDD開始前に、実装なしでテストケース設計を確定する。

## 必須ルール

1. 出力は `describe` と `it.todo` のみを使う
2. テストロジック（Arrange/Act/Assert、モック実装）は書かない
3. 各 `it.todo` の直前に、意図が分かる1行コメントを付ける
4. テスト名は日本語で「〜の時、〜であること」に統一する
5. 仕様が曖昧なら実装前にユーザーへ確認する

## 対象

- 関数・ロジック
- UIユースケース
- React Hook（カスタムHook含む）

## 禁止事項

- `it` / `test` に実行コードを書く
- モック前提のケースだけで構成する
- 仕様不明のまま断定ケースを大量に作る
