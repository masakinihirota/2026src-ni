# japanese-tech-writing スキル統合ガイド

## 統合完了

`japanese-tech-writing` スキルが `.agents/skills/japanese-tech-writing/` へインストールされました。

## クイックスタート

### 単独使用（推敲フェーズ）

```
/skill japanese-tech-writing

対象：[ドキュメント範囲]
焦点：[優先度High から段落と論証の構成をチェック]
```

### 推奨フロー（二段階推敲）

#### ステップ1：基本表記の整備

```
/skill writing-style

対象：[ドキュメント]
焦点：文体統一、表記ルール、簡潔さ
```

#### ステップ2：論証構造の厳密化

```
/skill japanese-tech-writing

対象：[ステップ1で整理されたドキュメント]
焦点：段落と論証、論証の厳密さ、LLM口調禁止
```

## 適用対象

**最適**：
- 技術書の章草稿
- 設計書・要件定義書
- 技術ブログ記事の推敲
- 障害分析報告書
- AI生成テキストのフィルタリング

**不適**：
- 初期段階の思考整理（完成度<50%）
- 広報・マーケティング文（硬すぎる可能性）
- リアルタイムチャット・カジュアルな記事

## セッション利用例

### シナリオ1：設計書の品質ゲート

```
ユーザー：「この設計書を推敲してください」

AI（あなた）：
1. writing-style スキルで表記統一
2. japanese-tech-writing で論証確認
3. 優先度High 3項目（段落構成、因果説明、LLM口調）をチェック
4. 改善提案を段落ごとに提示
```

### シナリオ2：技術記事の仕上げ

```
ユーザー：「長編記事を仕上げたい。LLMぽくないか確認」

AI（あなた）：
1. japanese-tech-writing の「LLM っぽい表現の禁止」セクション優先
2. 「正面から」「多角的に」などの空疎表現を検出
3. 代替表現を提案
4. 因果説明の機構が各所で示されているか確認
```

### シナリオ3：書籍原稿のリライト

```
ユーザー：「全3章を推敲。ただし6〜8時間は避けたい」

AI（あなた）：
1. 第1章を japanese-tech-writing で全項目チェック
2. パターンを特定（例：術語が章間で揺らいでいる）
3. 第2・3章へのチェックリストを提示
4. ユーザーが自己チェック、疑問な箇所だけAIへ確認
```

## 他スキルとの関係

| スキル | 役割 | 連携 |
|--------|------|------|
| `writing-style` | 基本表記・文体・表現ルール | 先に実行（ステップ1） |
| `japanese-tech-writing` | 論証構造・厳密さ・LLM口調 | 後に実行（ステップ2） |
| `code-review` | コード品質・セキュリティ | 技術ドキュメント最終チェック |
| `audit` | 品質スコアリング | 全体品質評価 |

## トラブルシューティング

**Q：スキルが呼び出せない**
→ `.agents/skills/japanese-tech-writing/` ディレクトリが存在し、`SKILL.md` が配置されているか確認してください。

**Q：出力が厳しすぎる**
→ 優先度 Low 項目は参考程度です。著者の意図を尊重してください。

**Q：human-written テキストに適用しても大丈夫か**
→ はい。特に「LLM っぽい表現の禁止」は人間の文章にも参考になります。
