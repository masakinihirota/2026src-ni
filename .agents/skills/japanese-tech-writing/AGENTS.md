# japanese-tech-writing スキルの Agent ルール

## 使用タイミング

以下のシーンで呼び出してください：

1. **技術書の章・節の推敲フェーズ**：下書き完成後、論証構造と厳密さをチェックする
2. **設計書・要件定義書の品質ゲート**：重要な決定事項を記述する際、論理の穴がないか確認する
3. **技術ドキュメント全体の リライト**：既存ドキュメントの論理性を向上させる
4. **AI生成テキストのフィルタリング**：LLM出力の空疎な表現を除去する推敲

## 前提条件

- **下書きが既に存在すること**（初期作成時ではなく推敲用）
- **対象範囲を明示すること**（例：「第3章」「段落5〜12」）
- **目的を明記すること**（例：「LLM口調を除去したい」「因果説明が不足していないか確認」）

## 呼び出し方法

### 他スキルから組み込む場合

AGENTS.md ファイルに以下を追記：

```yaml
---
name: [親スキル名]
skills:
  - japanese-tech-writing
  - [その他スキル]
---
```

### Prompt 指定する場合

```
まず writing-style で基本表記を整え、
その後 japanese-tech-writing で論証構造を推敲してください。

対象範囲：[セクション/ファイル]
焦点：[何に注目すべきか]
```

## 併用スキル（推奨フロー）

1. **下書き作成フェーズ**：特に指定なし
2. **基本表記の整備**：`writing-style` スキル
3. **論証構造の厳密化**：`japanese-tech-writing` スキル（本スキル）
4. **最終チェック**：`code-review` / `audit` スキル（技術文書の場合）

## スキル適用の観点

### 優先度 High（まず確認すべき項目）

- **段落と論証の構成**：一段落一トピック、段落先頭で前段落との論理関係を明示
- **論証の厳密さ**：因果関係に機構（なぜそうなるのか）を一文で示しているか
- **LLM っぽい表現**：空疎な修辞詞（「重要なのは」「正面から」「多角的に」）が混入していないか

### 優先度 Medium

- **読み手の負荷の管理**：不要な固有名詞・詳細が混入していないか
- **冗長の排除**：同じ主張の言い換え繰り返しがないか
- **視点と語り**：術語が途中で曖昧語に後退していないか

### 優先度 Low（仕上げ段階）

- **見出しの付け方**：見出しが対象や問いを特定できているか
- **演出の抑制**：不要な装飾的表現がないか
- **整形**：一文一行、脚注、コードブロックが適切か

## 出力の期待値

スキル適用後、以下を提供：

1. **チェック結果**：該当問題の位置・重大度・具体例
2. **修正提案**：段落単位または文単位での改写例
3. **パターン指摘**：全体に共通する論理ギャップ（例：全章で術語の定義が不足している）
4. **優先度付き改善プラン**：修正順序の推奨

## 注意事項

- 本スキルは **評価・提案型** で、修正を自動実行しません
- 推敲対象は **完成度50%以上の下書き** が想定です（初期段階は対象外）
- 「LLM っぽい表現」チェックは特に AI 出力文に効果的ですが、人間の文章にも適用できます
- 作者の「意図的な工夫」（必要な冗長性、戦略的な反復など）を尊重し、ただ規範に従わせるのではなく、理由を示して判断を委ねます
