# コーディング規約

## 命名規則
- 変数名・関数名はcamelCase
- 定数はUPPER_SNAKE_CASE
- 型/インターフェースはPascalCase
- boolean変数は `is`, `has`, `should` プレフィックスを使う

## 関数
- 1関数1責務。関数の行数は30行以内を目安
- 副作用のある関数とない関数を明確に分離する
- 引数は3つ以内。超える場合はオブジェクトで渡す

## 型安全性
- `any`型は原則禁止。必要な場合は`unknown`を使い、型ガードで絞り込む
- APIのリクエスト/レスポンスには必ず型定義を付ける
- nullableな値には`| null`を明示し、使用前にチェックする

## エラーハンドリング
- catch節でエラーを握りつぶさない。最低限ログ出力する
- ユーザー向けエラーメッセージに内部情報を含めない
- 非同期処理のエラーは必ずハンドリングする

## セキュリティ
- ユーザー入力をSQL, HTML, コマンドに直接埋め込まない
- パラメータ化クエリまたはORMを使用する
- レスポンスのContent-Typeを適切に設定する

## パフォーマンス
- ループ内でDBクエリやAPI呼び出しをしない（N+1問題）
- ループ外に出せる計算はループ外に出す
- マジックナンバーは定数として定義する
