# Code Review: Diff from Main (Standard)

このワークフローは、現在のブランチで追加・変更されたすべてのコード（コミット済みおよび未コミット）と `main` ブランチの差分を標準的なプロジェクト基準でレビューします。

## 手順

1.  **差分の取得**:
    - 新規ファイルが差分に含まれるよう、必要に応じて `git add -N .` を実行します。
    - PowerShellで `git diff $(git merge-base main HEAD)` を実行し、`main` との分岐点からの全差分を取得します。

2.  **標準レビュー実行**:
    - 以下の観点でコードをチェックします。
      - **機能とロジック**: バグやロジックの破綻はないか。要件に沿っているか。
      - **可読性・構造**: 命名規則（DBは `snake_case`、アプリ内は `camelCase`）やディレクトリ構造（コンポーネントのコロケーション、Barrel Exportの使用）に従っているか。
      - **最新仕様への追従**: Next.js 16 の非同期API (`await params`, `await cookies()`) が正しく使われているか。
      - **基本セキュリティ・品質**: `console.*` による不要なログ出力がないか（`logger.*`を使用）。意図しない `any` 型の使用がないか。
      - **フロントエンド設計**: Tailwind の装飾目的のグラデーションが使用されていないか、UIのコントラスト比が確保されているか。

3.  **レポート作成**:
    - 検出された問題点や改善の余地がある箇所をリストアップします。
    - 「来た時よりも美しく」のボーイスカウトルールに則り、気づいた軽微な改善点（リファクタリングなど）も提案し、具体的なコード例と共に提示します。
