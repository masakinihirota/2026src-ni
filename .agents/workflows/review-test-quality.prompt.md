# Test Review Workflow

テストコードを対象に「現在も必要か？」「不足しているテストはないか？」「通っていないテストはないか？」を検証し、結果を報告します。

## 手順

### 1. 全テストの実行と失敗テストの収集

// turbo
```powershell
pnpm test:run
```

✅ 失敗したテストがあれば、テストファイル名・テストケース名・エラー内容を一覧にまとめてください。

### 2. テストファイルの一覧を取得

// turbo
```powershell
Get-ChildItem -Path src -Recurse -Include *.test.ts,*.test.tsx,*.logic.test.ts,*.logic.integration.test.ts | Select-Object -ExpandProperty FullName
```

### 3. 各テストファイルのレビュー（以下の観点でチェック）

#### 3.1 不要テストの検出

以下に該当するテストを **「不要候補」** として報告してください：

- **テスト対象のコード（コンポーネント・関数・モジュール）が既に削除されている**
- **テスト対象のインターフェースが大幅に変更されており、テスト内容が実態と乖離している**
- **同一の振る舞いを検証する重複テストが存在する**（片方を統合または削除すべき）
- **モック過多で実装と無関係な検証しかしていない**（テストの価値がゼロに近い）
- **コメントアウトされたまま放置されているテストケース**

#### 3.2 不足テストの検出

以下に該当するケースを **「テスト追加候補」** として報告してください：

- **テストファイルが存在しないコンポーネントやロジックファイル**（`*.tsx` に対応する `*.test.tsx` がない等）
- **正常系しかなく、異常系・境界値・エラーハンドリングのテストがない**
- **アクセシビリティテスト（`vitest-axe`）が欠落しているUIコンポーネント**
- **認証・認可・セキュリティに関わるロジックでテストが不足している**
- **状態遷移（ローディング→成功/失敗）の検証が抜けている**

#### 3.3 テスト品質チェック

以下に該当する場合は **「改善候補」** として報告してください：

- **テストケース名が日本語でなく仕様が読み取れない**
- **AAA パターン（Arrange / Act / Assert）が守られていない**
- **`any` 型キャストでモックを作成している**
- **テスト内で実装の内部構造に過度に依存している**（リファクタリング耐性が低い）

### 4. レポート作成

以下の形式でレポートをまとめてください：

```markdown
## テストレビュー結果

### 📊 サマリー
- テストファイル数: XX
- テストケース総数: XX passed / XX failed / XX skipped
- 不要候補: XX 件
- 追加候補: XX 件
- 改善候補: XX 件

### 🔴 失敗テスト一覧
| ファイル | テストケース | エラー概要 |
|----------|------------|-----------|
| ... | ... | ... |

### 🗑️ 不要テスト候補
| ファイル | 理由 |
|----------|------|
| ... | ... |

### ➕ テスト追加候補
| 対象ファイル | 推奨テスト内容 | 優先度 |
|-------------|---------------|--------|
| ... | ... | High/Medium/Low |

### 🔧 改善候補
| ファイル | 指摘内容 |
|----------|---------|
| ... | ... |
```

### 5. 修正アクションの提案

レポート完了後、以下を提案してください：
- 失敗テストの修正方針（即時修正 or 後回し or 削除）
- 不要テストの削除リスト
- 優先度が高い追加テストの実装計画
