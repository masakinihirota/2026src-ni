# /boyscout ワークフロー

このワークフローは、プロジェクトルート直下に残った不要ログ、一時ファイル、AI生成の作業残骸を安全に整理するためのものです。
boyscout のうち、特に「不要ファイルの掃除」を独立した slash command として扱います。

## 目的

- ルート直下の不要ログや一時出力を削除する
- AI生成の一時ファイルを掃除して作業ツリーを見やすく保つ
- 判断に迷うファイルは削除せず、候補として報告する

## 実行方針

1. まず削除候補を列挙する
2. 明らかに不要なものだけ削除する
3. 判断に迷うファイルは残して報告する
4. 削除結果を最後に一覧で報告する

## 対象ファイル

`.agents/hooks/cleanup-temp-files.md` のパターンを優先して使用します。
特に以下を削除候補として確認します。

- `*.log`
- `*_output*.txt`
- `test-output.txt`
- `test_report.json`
- `analyze_tests.js`
- `*.backup`
- `apply_migration_*.js`
- `parse_tests.js`
- `.tmp*.json`
- `tmp`
- `*.csv`
- `*_AUDIT_RESULTS.md`

## 除外ルール

- `node_modules/`, `.next/`, `src/`, `docs/`, `scripts/`, `.agents/` 配下は対象外
- ルート直下のソースコードや設定ファイルは削除しない
- `page.tsx` などの迷子ソースは削除せず、必ず確認対象として報告する

## 手順

1. `.agents/hooks/cleanup-temp-files.md` を読み、対象パターンを確認する
2. ルート直下の候補ファイルを列挙する
3. 明らかに不要なファイルのみ削除する
4. 迷うものは「未削除候補」として報告する
5. 削除件数とファイル一覧を報告する

## 出力

- 削除したファイル一覧
- 未削除だが確認が必要なファイル一覧
- 今後 boyscout で継続的に掃除すべきパターンがあれば提案

## コマンド例

- `/boyscout`
- `/boyscout ルート直下だけ確認して安全なものだけ削除`
