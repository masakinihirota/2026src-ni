# /plan-today ワークフロー

このアプリの要件定義書と要件設計書（`anti-design` ワークツリー内の `0012 全体 -01要件定義書 -02設計書 -03テスト計画書` フォルダ）をみて開発予定を立てます。

- 今日のスケジュールの予定を立てます。

- 今日の分のtodoリストを作成します。

- 今週のスケジュールの予定を立てます。

- 今週の大まかなtodoリストを作成します。

- 以前のスケジュールを見て進捗具合を報告してください。
  MVPまでの完成を１００とした場合、全体でどの程度進んだのかを表示します。

- 残りの作業を表示します。

スケジュール報告ファイルの保存場所
/schedule_todo_list

一ヶ月以上古い報告書の保存場所
/schedule_todo_list/archive/old
期限が来たら移動させます。

補足:
- このワークフローで作成する「今日と今週の開発計画」「進捗報告」は `schedule_todo_list` を出力先とします。
- 古い報告書は `schedule_todo_list/archive/old` へ移動します。
- 既存の TODO 管理ルールと競合しないよう、保存先として `schedule` フォルダは使用しません。

## コマンド

- `/plan-today`
